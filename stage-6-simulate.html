Fin Procedures Workshop

Simulate & Stress Test

A Procedure isn't ready until it's been tested. Run these three scenarios to make sure yours handles every case correctly.

How to run a simulation In your Intercom workspace, open your Refund Request Procedure and click Simulate. You can provide a test customer and message, then watch Fin work through the steps.

Scenario 1: Happy Path — Eligible Refund

The most common case: customer has an order within the return window.

InputValue
Customer message"Hi, I'd like to return my recent order and get a refund."
Expected connector callsCustomer Lookup → Order Lookup → Refund Eligibility → Process Refund
Expected eligibility responseeligible: true
Expected Fin responseRefund confirmed with confirmation number
Pass criteria Fin calls all four connectors in order, processes the refund, and provides a confirmation number.

Scenario 2: Ineligible Order

Customer's order is outside the return window — Fin should explain and offer escalation.

InputValue
Customer message"I bought something a couple of months ago — can I still get a refund?"
Expected connector callsCustomer Lookup → Order Lookup → Refund Eligibility (no Process Refund)
Expected eligibility responseeligible: false, reason returned
Expected Fin responseExplains ineligibility, offers to escalate to support team
Pass criteria Fin takes the No branch, does NOT call Process Refund, and communicates the reason clearly.

Scenario 3: Order Not Found

Edge case — the customer's email returns no orders, or a wrong order ID is returned.

InputValue
Customer message"I want a refund but I can't find my order number."
Expected behaviourFin asks customer to verify their email or provide order number
Expected Fin responseAsks for clarifying information, does not proceed to eligibility check
If this scenario fails Go back to your Procedure and add a condition after Order Lookup: If orders list is empty → ask customer to confirm their email address or order number.
Checkpoint All three scenarios should pass before moving on. If any fail, revisit the relevant step in Stage 5 and adjust. Simulation is where you find the gaps — this is the process working as intended.
