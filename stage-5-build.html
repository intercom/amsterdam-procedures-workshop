Fin Procedures Workshop

Build the Procedure

Time to build. Follow each step carefully — by the end you'll have a working Refund Request Procedure.

Work in your shared workspace Make sure you're logged in to the shared Intercom workspace. You'll each be building your own copy of the Procedure.

Part 1: Create the Procedure

  1. Navigate to AI & Automation → Fin AI Agent → Procedures.

  2. Click New Procedure.

  3. Give it a name: Refund Request.

  4. In the Trigger field, describe when Fin should run this Procedure:

    "When a customer asks for a refund or wants to return an item."

Part 2: Add the Data Connector steps

  1. Click Add step → Call a Data Connector.

    Select Customer Lookup. Map the input: set email to the customer's conversation email.

  2. Add a second step: Call a Data Connector → Order Lookup.

    Map the input: set customer_id to the output of the previous step (customer_id).

  3. Add a third step: Call a Data Connector → Refund Eligibility.

    Map the input: set order_id to the most recent order from the Order Lookup results.

    Tip If the customer has multiple orders, Fin will use the most recent one by default. You can add a clarifying question step before this if you want the customer to confirm which order.

Part 3: Add the eligibility condition

  1. After the Refund Eligibility step, click Add step → Add condition.

  2. Set the condition: If eligible is true.

  3. In the Yes branch: add a step → Call a Data Connector → Process Refund. Map order_id from the Order Lookup output.

  4. After the Process Refund step, add a Send message step:

    "I've processed your refund for {{order_id}}. Your confirmation number is {{confirmation_id}}. You should see the refund within 3–5 business days."

  5. In the No branch: add a Send message step:

    "Unfortunately, your order isn't eligible for a refund. {{reason}}. If you think this is a mistake, I can connect you with our support team."

Part 4: Publish

  1. Review the full Procedure flow — make sure all steps are connected and inputs are mapped.

  2. Click Save, then Enable Procedure.

Checkpoint Your Procedure is now live in the shared workspace. Before moving to simulation, check that all four steps appear in the flow and the condition has both a Yes and No branch.
