Data Connectors

Four connectors are already set up in your shared workspace. Let's explore what each one does before we use them.

What is a Data Connector? A Data Connector is how Fin calls an external system during a Procedure. It defines the API endpoint to call, what data to send, and what data to extract from the response. Think of it as a pre-wired integration step.

Finding the connectors in your workspace

  1. In your shared Intercom workspace, go to AI & Automation → Fin AI Agent → Procedures.

  2. Click Data Connectors in the left navigation.

  3. You should see four connectors listed. Open each one to familiarise yourself with its configuration.

The four connectors

1. Customer Lookup

Retrieves a customer record using their email address.

FieldValue
MethodGET
Inputemail (from conversation context)
Returnscustomer_id, name, email

Example response:

{
  "customer_id": "cust_8821",
  "name": "Alex Johnson",
  "email": "alex@example.com"
}

2. Order Lookup

Retrieves the customer's recent orders.

FieldValue
MethodGET
Inputcustomer_id (from Customer Lookup)
ReturnsArray of orders: order_id, item, amount, date

Example response:

{
  "orders": [
    { "order_id": "ord_441", "item": "Blue Jacket", "amount": 89.99, "date": "2026-03-28" },
    { "order_id": "ord_398", "item": "Grey Hoodie", "amount": 54.00, "date": "2026-02-14" }
  ]
}

3. Refund Eligibility

Checks whether a specific order is eligible for a refund.

FieldValue
MethodGET
Inputorder_id (selected from Order Lookup)
Returnseligible (boolean), reason

Example responses:

// Eligible
{ "eligible": true, "reason": "Within 30-day return window" }

// Not eligible
{ "eligible": false, "reason": "Outside 30-day return window" }

4. Process Refund

Submits the refund and returns a confirmation.

FieldValue
MethodPOST
Inputorder_id
Returnssuccess (boolean), confirmation_id

Example response:

{ "success": true, "confirmation_id": "ref_7742" }
Mock endpoints These connectors call mock API endpoints — no real transactions take place. They're pre-configured to return realistic sample data so you can focus on building the Procedure, not on API setup.
Checkpoint Before moving on: open all four connectors in your workspace and confirm they're listed. You don't need to edit anything — just get familiar with the structure.
