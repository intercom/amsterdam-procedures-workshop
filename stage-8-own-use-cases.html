Your Own Use Cases

You've built the refund Procedure. Now apply the same process to a real workflow from your own business.

Switch workspaces For this stage, log in to your own Intercom workspace. Your facilitator will help if you need access.

The process you've learned

Every Procedure you build will follow the same pattern:

  1. Map your SOP on paper (trigger, data, decisions, outcomes)
  2. Identify the data connectors you need
  3. Build the Procedure step by step
  4. Add conditions for each decision point
  5. Simulate all scenarios before going live

Choosing your use case

Pick a process that:

  • Your team handles frequently
  • Requires looking up data from an external system
  • Has a clear trigger and defined outcomes
  • Doesn't require complex human judgement

Work time

Use the next ~30 minutes to:

  1. Go back to your Stage 2 worksheet. Apply the same questions to your chosen use case.

  2. Identify which data connectors you'd need to build (or check if any already exist in your workspace).

  3. Start building the Procedure in your own workspace — get as far as you can.

You don't need to finish today The goal is to start the process and get unstuck at the point where most people stop: mapping the SOP. Even a partial Procedure with placeholder connectors is valuable progress.
