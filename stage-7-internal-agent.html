Advanced: Internal Agent

How do Procedures change when Fin is working as an internal agent — helping your support team rather than your customers?

What is Internal Agent?

Internal Agent is Fin working inside your team's inbox, assisting human agents rather than talking directly to customers. Instead of resolving conversations autonomously, Fin surfaces relevant information, suggests responses, and executes actions on behalf of the agent.

How Procedures work with Internal Agent

The same Procedure you built today can be repurposed for Internal Agent with a few adjustments:

Customer-facing ProcedureInternal Agent Procedure
TriggerCustomer sends a messageAgent opens a conversation or requests a lookup
Data connectorsSame connectors, same dataSame connectors, same data
OutputFin replies directly to customerFin surfaces data and a suggested response to the agent
ActionsFin processes refund autonomouslyAgent reviews and approves before action is taken

When to use Internal Agent vs. autonomous Procedures

Use autonomous Procedures when:

  • The process is well-defined and low-risk
  • The outcome is reversible or confirmable
  • Volume is high enough to justify full automation

Use Internal Agent when:

  • Human judgement is still needed before taking action
  • The process handles sensitive or high-value cases
  • You want to build team confidence before going fully autonomous
A useful starting point Many teams start with Internal Agent — Fin assists agents on refunds — before moving to fully autonomous Procedures once they're confident in the data and outcomes. It's a lower-risk way to get value from your Procedures earlier.
